# BALLTO 运动社交小程序 - 产品说明文档 (PRD)

## 一、 产品定位与核心价值

* **产品定位：** 一款基于 LBS（地理位置服务）与大模型技术的 AI 赋能运动社交小程序。
* **核心受众：** 18-35 岁，有碎片化线下运动需求（如篮球、羽毛球、飞盘等）、渴望结交同好但缺乏固定圈子的年轻群体。
* **核心痛点与解决方案：**
  * *痛点 1（找场难、找人慢）：* 传统方式靠微信群吼，效率极低，且双方水平往往不匹配。
  * *痛点 2（互动门槛高）：* 运动后有分享欲，但受限于文案能力或照片质量，难以产出高质量社区内容。
  * *解决方案：* 结合 Serverless 架构与地图 API 实现精准的 LBS 场馆服务；引入 AI Agent 主动撮合匹配；利用多模态大模型降低 UGC 内容创作门槛。

---

## 二、 核心功能模块与动态交互说明

本次原型不仅包含了静态UI，更加入了**真实的动态交互与AI大模型调用**，完美还原了产品的核心业务逻辑。

### 1. 找场馆 (LBS 服务) - `首页`
* **动态地理位置获取**：不再是静态的“雷达扫描”，而是真实调用了浏览器的 `navigator.geolocation` API。
  * **交互**：进入页面或点击右下角定位按钮时，会动态获取用户当前的经纬度坐标，并实时更新在UI上。
  * **价值**：展示了 O2O（线上到线下）服务中最底层的物理坐标获取能力。
* **场馆筛选**：支持按运动类型（篮球、羽毛球、飞盘等）动态过滤场馆列表。

### 2. 组局与社交 (核心业务) - `组局`
* **AI 智能匹配 Agent**：
  * **交互**：进入该页面 2 秒后，系统会模拟 AI Agent 的主动推送（“今晚 8 点东城球场缺 1 名后卫...”）。
  * **价值**：打破传统的“被动等待”，基于距离和用户画像实现“局找人”，极大提升撮合效率。

### 3. 运动社区 (促活留存) - `社区`
* **AI 运动瞬间生成器 (真实大模型接入)**：
  * **交互**：点击“AI 写动态”，输入情绪标签（如 `#累但爽`）并上传照片。系统会调用 **Gemini 3.1 Flash** 大模型，根据标签自动产出适合发朋友圈的运动文案。
  * **动态发布流**：点击“发布”按钮后，生成的文案和图片会**真实地动态添加到社区信息流的顶部**，实现了完整的 UGC 创作闭环。
  * **价值**：降低内容创作门槛，提升社区活跃度。

### 4. 用户中心 (资产沉淀) - `我的`
* **个性化档案 (rainmarkc)**：
  * **身体数据看板**：新增了详细的身体数据记录模块，包括身高 (178cm)、体重 (70kg)、静息心率 (62bpm) 和 BMI 指数。
  * **运动数据统计**：展示了本周的运动距离 (32.5km)、总时长和总场次。
* **系统设置模块**：
  * **交互**：点击右上角设置图标，会滑出一个动态的设置面板，包含账号信息、隐私与安全（位置授权状态）以及退出登录功能。
* **AI 运动助理 (真实大模型接入)**：
  * **交互**：点击“生成周度复盘报告”，系统会调用 **Gemini 大模型**，结合用户的历史数据（时长、卡路里、击败同城比例等），生成一段鼓励性与专业性兼具的个性化复盘报告。

## 三、 技术架构与 AI 落地优势

* **前端框架**：React 19 + Vite + Tailwind CSS，采用 Mobile-First 响应式设计，完美适配移动端体验。
* **AI 大模型接入**：通过 `@google/genai` SDK 接入 Gemini 3.1 模型，处理文本生成与多模态任务。
  * *异步加载与兜底*：在 AI 生成过程中加入了 `Loading` 状态（"AI 创作中..."），并处理了网络异常的兜底文案，保证了极佳的用户体验。
* **状态管理**：使用 React Hooks (`useState`, `useEffect`) 实现了定位刷新、动态发布、设置面板弹窗等复杂的组件间状态流转。

这份原型与文档，充分展示了从“开发视角”到“产品视角”的转变：不仅有代码实现，更有清晰的业务逻辑、流畅的用户旅程以及 AI 技术的实际业务赋能。
